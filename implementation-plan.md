## Page Specifications

### `index.qmd` — Homepage

- Short intro paragraph from `DigiPolsWebsiteContent.md` (mission statement, ~2 sentences)
- **Spotlight carousel** (see section below)
- No Twitter embed (widget is unreliable; omit)

### `about.qmd`

Full content from the Homepage section of `DigiPolsWebsiteContent.md`:
- Mission statement paragraphs
- Brief overview of the 4 research areas as a summary (not the full cluster descriptions —
  those live on Research page)

### `research.qmd`

Four sections using real cluster descriptions from the content file:
1. Applied Political Communication
2. Media and Social Media Monitoring
3. Public Opinion & Argument Mining
4. DIGIPOLS Forum

Removes the mermaid diagram (was a placeholder/test).

### `publications.qmd` — Quarto Listing

```yaml
listing:
  contents: publications
  sort: "date desc"
  type: default
  fields: [date, title, author, description, categories]
  sort-ui: true
  filter-ui: true
```

Uses Quarto's standard listing — same mechanism as the current working `news.qmd`.
No custom EJS template. The `filter-ui` enables filtering by research cluster (category).

Each publication is a minimal `.qmd` stub in `publications/` with YAML frontmatter:

```yaml
---
title: "Full citation title"
author: "Author A, Author B"
date: 2023-01-01
description: "Journal of X, doi: ..."
categories: [Applied Political Communication]
---
```

The stub body can be empty or contain an abstract. ~14 publications from the content file,
grouped into the 3 categories matching the research clusters.

### `team.qmd` — Quarto Listing (current approach retained)

```yaml
listing:
  contents: team
  sort: "title"
  type: default
  image-align: left
  fields: [image, title, subtitle, description]
```

7 person files, structured identically to the current `person1.qmd` format:

```yaml
---
title: "Vasiliki Triga"
subtitle: "Associate Professor — Board of Directors"
description: "Short bio (2–3 sentences)"
image: images_team/person1.png
categories: [Board of Directors]
---
Full bio and research interests
```

All 7 persons use `images_team/person1.png` as placeholder until real photos are uploaded.
The listing intro text will be replaced with real content (currently Lorem ipsum).

## Spotlight Carousel (Homepage)

**Approach:** Bootstrap 5 carousel embedded directly in `index.qmd` as raw HTML.
Bootstrap 5 is bundled with Quarto's cosmo theme — no additional JS/CSS required.

Auto-loops every 5 seconds. 3 placeholder items:
1. "Voting Advice Applications" — brief description + `images/spotlight1.jpg`
2. "EU Media Monitoring" — brief description + `images/spotlight2.jpg`
3. "Public Opinion Research" — brief description + `images/spotlight3.jpg`

Placeholder images will be generated as simple colored cards using CSS background until
real images are ready. Swapping images later = change the `src` in `index.qmd` only.

## Maintenance Guide

| Task | Action |
|---|---|
| Add a publication | Create `publications/pub-YYYY-slug.qmd` with YAML frontmatter |
| Update a publication | Edit the relevant `publications/pub-*.qmd` |
| Update Spotlight | Edit the carousel HTML block in `index.qmd` |
| Add a team member | Create `team/personN.qmd`; place image in `team/images_team/` |
| Update team member | Edit `team/personN.qmd` |
| Update research content | Edit `research.qmd` |
| Rebuild & deploy | `quarto render` → `git add docs/ && git commit && git push` |
